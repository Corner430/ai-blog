本文详细介绍如何使用 x-ui 面板搭建多种协议的代理节点，包括 vmess+ws、vless+vision+tls、vless+vision+reality、hysteria2，以及在纯 IPv6 VPS 上使用 sing-box 搭建节点的方法。

## 1 x-ui 面板安装与配置

[FranzKafkaYu/x-ui](https://github.com/FranzKafkaYu/x-ui) 是一个支持多协议、多用户的 xray 面板，提供 Web 界面管理。

### 1.1 Docker 安装

```shell
docker pull enwaiax/x-ui:alpha-zh
```

### 1.2 使用 docker-compose 运行

创建 `docker-compose.yml` 文件：

```yml
services:
  xui:
    image: enwaiax/x-ui:alpha-zh
    tty: true
    container_name: xui
    volumes:
      - $PWD/db/:/etc/x-ui/
      - $PWD/cert/:/root/cert/
    restart: unless-stopped
    network_mode: host
```

启动容器：

```shell
docker compose up -d
```

### 1.3 Web 界面配置

1. 访问 `http://你的服务器IP:54321`，默认用户名密码均为 `admin`
2. 进入面板设置，修改监听端口和密码（**强烈建议修改默认密码**）
3. 切换 xray 版本为 `1.8.3`

## 2 vmess+ws 节点搭建

vmess+ws 是最通用的协议组合，兼容性好，支持 CDN 中转。

### 2.1 配置步骤

1. 备注：`vmess+ws`
2. 协议：`vmess`
3. 添加一个用户
4. 网络：`ws`
5. 复制 **id 的第一段** 追加到**路径**后面
6. 点击添加

## 3 vless+vision+tls 节点搭建

vless+vision+tls 提供更好的安全性和伪装能力。

### 3.1 创建入站

1. 备注：`vless+vision`
2. 添加用户
3. 打开 `tls` 开关
4. flow：`xtls-rprx-vision`
5. 将域名 IP `1.1.1.1` 改为 `1-1-1-1.nip.io` 的形式（nip.io 是一个免费的通配符 DNS 服务）

### 3.2 申请 TLS 证书

安装证书工具：

```shell
curl https://get.acme.sh | sh; apt install socat -y || yum install socat -y; ~/.acme.sh/acme.sh --set-default-ca --server letsencrypt
```

申请证书（记得将 IP 替换为你自己的），三种方式任选其一，申请失败则更换方式：

```shell
# 申请证书方式1：
~/.acme.sh/acme.sh  --issue -d 1-1-1-1.nip.io --standalone -k ec-256 --force --insecure
# 申请证书方式2：
~/.acme.sh/acme.sh --register-account -m "${RANDOM}@chacuo.net" --server buypass --force --insecure && ~/.acme.sh/acme.sh  --issue -d 1-1-1-1.nip.io --standalone -k ec-256 --force --insecure --server buypass
# 申请证书方式3：
~/.acme.sh/acme.sh --register-account -m "${RANDOM}@chacuo.net" --server zerossl --force --insecure && ~/.acme.sh/acme.sh  --issue -d 1-1-1-1.nip.io --standalone -k ec-256 --force --insecure --server zerossl
```

### 3.3 安装证书

```shell
~/.acme.sh/acme.sh --install-cert -d 1-1-1-1.nip.io --ecc --key-file /etc/x-ui/server.key --fullchain-file /etc/x-ui/server.crt

# 输出如下
[Thu May 16 12:13:43 CST 2024] Installing key to: /etc/x-ui/server.key
[Thu May 16 12:13:43 CST 2024] Installing full chain to: /etc/x-ui/server.crt
cat: /root/.acme.sh/10-0-0-4.nip.io_ecc/fullchain.cer: No such file or directory
```

### 3.4 填写证书路径

- **密钥文件路径**：`/etc/x-ui/server.key`
- **公钥文件路径**：`/etc/x-ui/server.crt`

点击添加完成配置。

![vless+vision+tls配置](https://cdn.jsdelivr.net/gh/Corner430/Picture/images/20240516121901.png)

> **性能提示**：导入客户端后，将地址由 `1-1-1-1.nip.io` 改为 `1.1.1.1` 形式的 IP 地址，可以加快连接速度。

## 4 vless+vision+reality 节点搭建

Reality 协议无需申请证书，伪装能力更强。

### 4.1 配置步骤

1. 备注：`vless+vision+reality`
2. 端口：`443`
3. 添加用户
4. 打开 `reality` 开关
5. flow：`xtls-rprx-vision`
6. **目标网站**的网址部分改为：`1.1.1.1`，保留端口号
7. 使用 **id 的第一段** 覆盖**可选域名**，并追加 `.com` 到可选域名的尾部
8. 点击添加

![vless+vision+reality配置](https://cdn.jsdelivr.net/gh/Corner430/Picture/images/20240516122437.png)

> 如果想增强伪装效果，可以点击**目标网站**后面的刷新按钮来更换伪装站点。**不要使用 speedtest** 作为伪装站点！修改后点击保存即可。

## 5 vmess 无 CDN

最简单的配置方式，直连模式：

直接在 x-ui 面板中点击添加入站，选择 `vmess` 协议，创建节点并添加用户即可。

> 这种方式不经过 CDN，速度更快但 IP 容易被封。

## 6 Hysteria2 节点搭建

Hysteria2 基于 QUIC 协议，能显著提升网络质量较差的 VPS 的实际速度。

参考资料：

- [Hysteria2 节点搭建教程](https://www.youtube.com/watch?v=CXj-ID33MhU)
- [Hysteria GitHub](https://github.com/apernet/hysteria?tab=readme-ov-file)
- [Hysteria 中文文档](https://v2.hysteria.network/zh/)

> **申请证书需要放行 80、443 端口**

### 6.1 服务器安装与管理

```shell
#一键安装Hysteria2
bash <(curl -fsSL https://get.hy2.sh/)

#生成自签证书
openssl req -x509 -nodes -newkey ec:<(openssl ecparam -name prime256v1) -keyout /etc/hysteria/server.key -out /etc/hysteria/server.crt -subj "/CN=bing.com" -days 36500 && sudo chown hysteria /etc/hysteria/server.key && sudo chown hysteria /etc/hysteria/server.crt

#启动Hysteria2
systemctl start hysteria-server.service
#重启Hysteria2
systemctl restart hysteria-server.service
#查看Hysteria2状态
systemctl status hysteria-server.service
#停止Hysteria2
systemctl stop hysteria-server.service
#设置开机自启
systemctl enable hysteria-server.service
#查看日志
journalctl -u hysteria-server.service
```

### 6.2 服务器配置文件

```shell
cat << EOF > /etc/hysteria/config.yaml
listen: :443 #监听端口，可修改

#使用CA证书
#acme:
#  domains:
#    - a.com #你的域名，需要先解析到服务器ip
#  email: test@sharklasers.com

#使用自签证书
#tls:
#  cert: /etc/hysteria/server.crt
#  key: /etc/hysteria/server.key

auth:
  type: password
  password: 123456 #设置认证密码

masquerade:
  type: proxy
  proxy:
    url: https://bing.com #伪装网址，可修改
    rewriteHost: true
EOF
```

配置说明：

- 如果有域名，取消 `acme` 部分的注释，填入域名和邮箱
- 如果没有域名，取消 `tls` 部分的注释，使用自签证书
- `password` 必须修改为自定义的强密码

### 6.3 客户端配置文件

```js
server: ip:443
auth: 123456

bandwidth:
  up: 20 mbps
  down: 100 mbps

tls:
  sni: a.com
  insecure: false #使用自签时需要改成true

socks5:
  listen: 127.0.0.1:1080
http:
  listen: 127.0.0.1:8080
```

## 7 纯 IPv6 节点搭建（sing-box）

部分 VPS 只有 IPv6 地址，可以通过 WireGuard + WARP 转为双栈网络后再搭建节点。

### 7.1 环境准备

```shell
# 查看内核版本
uname -a

apt update && apt upgrade
```

### 7.2 安装 WireGuard

```shell
# 安装 wireguard 内核模块
apt install wireguard-dkms -y
# 亲测这一条并不好用，但会帮你扫清许多依赖的问题，转到手动编译
# 下载并解压wireguard-tools源码并解压
wget https://git.zx2c4.com/wireguard-tools/snapshot/wireguard-tools-1.0.20210914.tar.xz
tar -xf wireguard-tools-1.0.20210914.tar.xz
# 手动编译其源码
cd wireguard-tools-1.0.20210914/src
make && make install
# 验证是否安装成功
wg -h
wg-quick -h
```

> `apt install wireguard-dkms` 可能不会完全成功，但它会安装许多必要的依赖包，之后通过手动编译 wireguard-tools 源码来完成安装。

### 7.3 配置 WARP 双栈

```shell
# 重启vps
reboot

touch /etc/wireguard/warp.conf
wget -N https://gitlab.com/fscarmen/warp/-/raw/main/menu.sh && bash menu.sh [option] [lisence/url/token]
# 此时菜单中选择转为双栈网络
```

### 7.4 使用 sing-box 搭建节点

```shell
bash <(wget -qO- https://raw.githubusercontent.com/fscarmen/sing-box/main/sing-box.sh)
# 一路傻瓜操作
```

按照脚本提示进行配置即可完成节点搭建。
