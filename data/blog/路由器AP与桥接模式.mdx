## 概述

对于配备AP（接入点）模式的路由器，可以直接启用该功能。如果路由器不带原生AP模式，则需采用桥接模式（手动AP方式）扩展无线覆盖。本文介绍这两种配置方法及常见问题处理。

---

## 基础网络配置

### WAN 和 LAN 口设置

配置AP或桥接模式时，首先进行以下基础设置：

1. **WAN 口处理**：保持断开状态，不要做任何配置
2. **LAN 口 IP 设置**：需要手动配置不同的 IP 地址来区分各个路由

**多级路由 IP 配置示例**：

- 一级路由（主路由）：`192.168.1.1`
- R2S 二级路由：`192.168.1.2`
- TP-Link 二级路由：`192.168.1.3`
- 其他路由：`192.168.1.4`、`192.168.1.5` 等

## AP 模式配置（有原生AP功能）

### 关闭辅助路由的 DHCP

所有除了一级主路由外的路由器都要关闭 DHCP 功能，防止 IP 分配冲突：

![tp_dhcp](https://cdn.jsdelivr.net/gh/Corner430/Picture/images/tp_dhcp.png)

### 无线设置

- 挑选一个路由器开启无线，其余路由的无线全部关闭
- 这样可以避免无线干扰和重复广播

## 无线桥接模式（手动AP）

对于不支持原生AP模式的路由器，可以使用桥接模式实现级联：

![20230514162847](https://cdn.jsdelivr.net/gh/Corner430/Picture/images/20230514162847.png)

### 配置步骤

1. 连接到主路由的有线网络或 WiFi
2. 进入路由器管理页面（通常 192.168.1.1）
3. 找到「无线设置」或「WiFi 设置」选项
4. 启用「桥接模式」或「中继模式」
5. 选择要连接的主路由 WiFi 名称
6. 输入主路由的 WiFi 密码
7. 应用设置并重启路由器

配置完成后，连接到该路由器的设备将通过有线或无线连接到主路由的网络。

## 故障排查

### ⚠️ 重要提示

> 这样配置会导致所有路由器的 WAN 口都失效。这是正常现象，因为 WAN 口已被禁用。

### 常见问题

1. **连接不稳定**：确保各路由器的信号覆盖无重叠区域太大
2. **IP 冲突**：验证所有辅助路由的 DHCP 都已关闭
3. **桥接连接失败**：检查主路由的 WiFi 密码是否正确，尝试重启路由器

### 重启所有路由器

> **重要**：重启所有路由器（包括主路由和辅助路由）！！！

配置完成后，建议依次重启所有路由器以应用设置：

```bash
# 通过 SSH 重启（需启用 SSH）
ssh admin@192.168.1.x
reboot
```

或通过管理页面点击「重启系统」按钮。

---

## 总结

- **AP 模式**：适合原生支持的路由器，配置简单
- **桥接模式**：通用解决方案，适用于大多数路由器
- **关键点**：LAN IP 不同、DHCP 关闭、只开一个无线、记得重启

选择合适的方案为家庭网络添加覆盖，享受更好的 WiFi 体验。
